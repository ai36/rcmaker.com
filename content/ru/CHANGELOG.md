# История изменений

## [0.0.8] - 2025-06-19

+ Добавлена поддержка локальных файлов конфигурации. Теперь вы можете задать настройки расширения в файле `.rcmakerrc.json` (рекомендуется размещать рядом с `package.json` проекта). Это позволяет хранить разные настройки для каждого проекта без необходимости изменять глобальные при каждом переключении.

## [0.0.7] - 2025-06-16

+ Исправлена ошибка, при которой создавался файл `types.ts` вместо `index.ts`

## [0.0.6] - 2025-05-27

+ Добавлен новый литерал для шаблонов: `${componentBaseFileName}` — имя файла компонента без расширения `.jsx` или `.tsx`
+ Шаблон файла реэкспорта `index.js` (или `index.ts`) изменён: строка экспорта теперь формируется на основе имени файла компонента без расширения
+ Шаблон компонента по умолчанию теперь редактируется прямо в настройках VS Code. При создании компонента шаблон `_default` всегда отображается первым и выбирается автоматически при нажатии Enter
+ Исправлена ошибка, при которой index-файл создавался с расширением `.js` при включённой настройке «Use TypeScript»

## [0.0.5] - 2025-05-04

+ Быстрое исправление

## [0.0.4] - 2025-05-04

+ Добавлен favicon
+ Добавлены подсказки в настройках для подставляемых переменных в шаблонах

## [0.0.3] - 2025-05-03

+ Добавлена возможность удобного редактирования шаблонов `index.js`, `component.test.jsx/tsx` и `types.ts`
+ Шаблоны компонентов могут быть в форматах jsx, tsx, js и ts
+ Добавлена функция создания barrel-файла (реэкспорта) в текущей папке по шаблону или добавления реэкспорта в `index.js` родительской папки. В настройках можно указать, нужно ли создавать `index.js`, задать шаблон, и выбрать, где создавать файл — в папке компонента или родительской директории
+ Добавлен новый стиль именования — **snake_case**

## [0.0.2] - 2025-04-13

+ Оставлен один шаблон тестов. Остальные можно создать самостоятельно
+ В шаблоне доступен литерал `${camelCase}` — имя созданного компонента в camelCase
+ Обновлена документация

## [0.0.1] - 2025-04-12

+ Начальная версия
